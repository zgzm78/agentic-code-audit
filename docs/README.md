# 文档目录

本目录只保留当前仍有维护价值的项目文档。历史阶段方案、重复架构说明和导出 HTML 已清理。

| 文档 | 内容 |
|---|---|
| [使用说明.md](使用说明.md) | 本地配置、Docker 启动、CLI、报告导入、开发测试 |
| [架构.md](架构.md) | 漏洞挖掘、验证和证据生成相关的核心架构 |
| [漏洞挖掘质量优化规划.md](漏洞挖掘质量优化规划.md) | 后续优化计划，重点是候选质量、切片分析和代码清理 |
| [测试.md](测试.md) | 已测试项目和结果汇总 |
| [成果.md](成果.md) | 可用于汇报展示的代表性漏洞案例 |
| [技术调研报告.md](技术调研报告.md) | 大模型智能体代码审计与漏洞研究调研 |

维护原则：

1. 新增开发计划优先合并到 `漏洞挖掘质量优化规划.md`，避免继续产生多份阶段性方案。
2. 架构变化优先更新 `架构.md`，不要新增重复的系统架构文档。
3. 使用、启动、导入、测试命令统一维护在 `使用说明.md`。
4. 大体量导出文件不要放入 `docs`，需要提交时放在课程提交目录或 release artifact。
